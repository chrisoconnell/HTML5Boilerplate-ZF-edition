Bringing the awesome-ness of HTML5 Boilerplate and the Zend Framework together!

Wouldn’t it be nice it you could just type zf create project MyProject into the command line and have all of the HTML5 Boilerplate goodness magically injected into your new Zend project? Well now you can!

This project uses Zend_Tool to work its magic so any version of Zend Framework 1.8 or higher should work. Just download the files and follow the instructions below.

Download

You can download this project in either zip or tar formats.

You can also clone the project with Git by running: 

$ git clone git://github.com/chrisoconnell/HTML5Boilerplate-ZF-edition

Install

  1. Find the location of your Zend Framework install. Mine is at "/usr/local/zend/share/ZendFramework" but yours could be different.
  2. Now go to "library/Zend/Tool/" so your final path should be "/path/to/ZendFramework/library/Zend/Tool/"
  3. Make a backup of the Project directory (just in case).
  4. Replace the "Project.php" file in the "Project/Provider" directory with the one from HTML5Boilerplate-ZF-edition.
  5. Copy every file beginning with "H5bp" into "Project/Context/Zf"

So if you are doing everything from the command line, it will look like: 

$ cd /path/to/ZendFramework/library/Zend/Tool/
$ cp -r Project Project-old
$ cp /path/to/HTML5Boilerplate-ZF-edition/Project.php Project/Provider/
$ cp /path/to/HTML5Boilerplate-ZF-edition/H5bp* Project/Context/Zf/

Thats it! Now anytime you create a new Zend Framework project, it will automatically have the HTML 5 Boilerplate template built in. You do not need to run zf enable layout as it is part of the new create project command.

Project Files

Files to be copied to library/Zend/Tool/Project/Context/Zf

H5bpJsLibsDirectory.php
Creates the public/js/libs directory.
H5bpJsLibsJqueryFile.php
Creates the public/js/libs/jquery-1.6.2.min.js file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpJsLibsModernizrFile.php
Creates the public/js/libs/modernizr-2.0.6.min.js file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpJsPluginsFile.php
Creates the public/js/plugins.js file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpJsSriptFile.php
Creates the public/js/script.js file.
Contents — what little there are — are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpPublicStylesheetsDirectory.php
Creates the public/css directory.
H5bpPublicStylesheetFile.php
Creates the public/css/styles.css file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpPublicRobotsFile.php
Creates the public/robots.txt file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpHtaccessFile.php
Creates the public/.htaccess file.
Includes all the content from the HTML5 Boilerplate 2.0 version plus the rules defined in a standard Zend Framework project.
H5bpPublicHumansFile.php
Creates the public/humans.txt file.
Contents are unchanged from the HTML5 Boilerplate 2.0 version.
H5bpPublicImagesDirectory.php
Creates the public/img directory.
H5bpLayoutScriptFile.php
Creates the application/layouts/scripts/layout.phtml file.
Intergrated standard Zend view helpers into layout inplace of hardcoding meta data and css/js files.
Layout script works in conjunction with the bootstrap file to define meta data and css/js files.
The resulting source html will look almost identical to HTML 5 Boilerplate 2.0 index.html file. The biggest difference will be the comments which had to be grouped together.
H5bpViewScriptFile.php
Creates the application/views/scripts/index/index.phtml file
Creates the application/views/scripts/error/error.phtml file — modified to be a blend of the HTML 5 Boilerplate 2.0 404.html file and a standard Zend Framework project error.phtml file.
H5bpBootstrapFile.php
Creates the application/Bootstrap.php file.
Added _initViewSettings resource to set layout meta data and define css/js files.
H5bpApplicationConfigFile.php
Creates the application/configs/application.ini file.
Similar to the application.ini file created by the standard Zend project.
Added settings for doctype and charset.
Added setting for baseUrl but left commented out as Zend should be able to auto detected if your VHOST is set properly.

Files to be copied to library/Zend/Tool/Project/Provider

Project.php
This is the file that sets the file directory structure and calls all of the helper php files above.
If you would like to add or remove files from this project, this file will need to be modified accourdingly.
Manifest.php
This file has not been modified from the original Zend Framework version for this project.
However, if you would like to rename the Project.php file — to say H5bpProject.php &mdash then you would have to modify this file.
The reason for doing such would be if you wanted to have the command line 
zf create project myProject
create a standerd Zend project. And have the command line 
zf create h5bpProject myH5bpProject
create this project.

What’s not included?

I’ve worked very hard to make this project as complete and thorough as possible. Even still, there were a few files contained in the HTML 5 Boilerplate distribution that I have left out.

Files from HTML 5 Boilerplate not included in this project

apple-touch-icons & favicon
This was because of technical reasons as I don’t know how to include non-text files using Zend_Tool.
If you know how to do this please contribute to this project.
build files
Contain exe files which I don’t know how to include.
test files
I feel this project will be better served by more Zend Framework specific tests. While the project does create the test folder from a standard Zend Framework project, I feel it is incomplete.
Will impove the test files in future versions of this project.
